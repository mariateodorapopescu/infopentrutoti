Algoritmi cu cifrele unui numar

Acest tip de algoritmi lucreaza cu urmatoarele expresii:
n%10=ultima cifra a lui n
n/10= numarul n din care s-a eliminat ultima cifra
Algoritmii uzuali sunt:
- aflarea sumei cifrelor unui numar
- aflarea numarului de cifre pe care le are un numar
- aflarea oglinditului unui numar
- suma cifrelor unui numar daca n=732848, atunci suma cifrelor sale este 7+3+2+8+4+8=32 Pentru a afla suma cifrelor parcurgem pasii:
aflam ultima cifra din n o adunam la o variabila s initializata cu 0, eliminam cifra adunata din n repetam pasii anteriori pana cand n=0 in pseudocod, avem urmatorul algoritm:
citeste n
s=0
cat timp n?0 executa
c=n%10
s=s+c
n=n/10
scrie s
- numarul cifrelor unui numar: daca n=732848, atunci numarul de cifre este 6 Pentru a afla numarul de cifre parcurgem pasii: creste cu 1 o variabila contor nr initializata cu 0 eliminam cifra adunata din n repetam pasii anteriori pana cand n=0 in pseudocod, avem urmatorul algoritm:
citeste n
nr=0
cat timp n?0 executa
nr=nr+1
n=n/10
scrie nr
- aflarea oglinditului unui numar: daca n=732848, atunci oglinditul sau este 848237 Pentru a afla oglinditul parcurgem pasii: aflam ultima cifra din n o adaugam la sfarsitul unei variabile inv initializata cu 0, eliminam cifra adunata din n repetam pasii anteriori pana cand n=0 pentru exemplul dat avem urmatoarele operatii: n=732848 deci c=8 ; inv=8; n=73284
n=73284 deci c=4; inv=84; n=7328
n=7328 deci c=8; inv=848; n=732 //cum ajung de la 84 la 848 in calculul lui inv? ce operatie se foloseste?
n=732 deci c=2; inv=8482; n=73
n=73 deci c=3; inv=84823; n=7
n=7 deci c=7; inv=848237; n=0
n=0 => ne oprim din structura repetitiva.
Ca sa adaugam la sfarsitul lui inv o cifra se foloseste operatia inv=inv*10+n%10
in pseudocod, avem urmatorul algoritm:
citeste n
inv=0
cat timp n?0 executa
c=n%10
inv=inv*10+c //aici se adauga c la sf. lui inv.
n=n/10
scrie inv
Observatie: un numar este palindrom daca este egal cu oglinditul sau (exemplu: 11, 121, 3, 36763, etc.)
